A module is a self-contained feature package that plugs into the Mapas Culturais core. Each module is a PHP class extending `MapasCulturais\Module`. Modules register their behavior — hooks, controllers, entities, job types — inside the `_init()` method, which the framework calls automatically during application boot.

## Module structure

```
src/modules/MyModule/
  Module.php       → Main module class (must extend MapasCulturais\Module)
  Entities/        → Doctrine entities added by this module (optional)
  Controllers/     → Controllers added by this module (optional)
  Jobs/            → Background job types (optional)
  views/           → View overrides or additions (optional)
```

## Writing a module

```php
<?php
namespace MapasCulturais\Modules\MyModule;

use MapasCulturais\App;

class Module extends \MapasCulturais\Module
{
    function _init(): void
    {
        $app = App::i();

        // Access module configuration
        $setting = $this->_config['my_setting'] ?? 'default';

        // Register a hook
        $app->hook('entity(Agent).save:after', function() use ($app, $setting) {
            // $this is the Agent entity that was just saved
            $app->log->info("Agent saved: {$this->name}");
        });

        // Register a controller
        $app->registerController('my-module', Controllers\MyController::class);

        // Register a background job type
        $app->registerJobType(new Jobs\MyJob(Jobs\MyJob::SLUG));
    }

    function register(): void
    {
        // Called before _init(). Use to register definitions,
        // evaluation methods, or other early-boot items.
    }
}
```

<Note>
  Module entities placed in `Entities/` are automatically discovered by Doctrine. No additional ORM configuration is needed.
</Note>

## Registering a module

Add the module to the `plugins` key in your configuration file:

```php
// config/config.php or dev/config.d/0.main.php
return [
    'plugins' => [
        'MyModule' => [
            'namespace' => 'MapasCulturais\Modules\MyModule',
            'my_setting' => 'custom_value',
        ],
    ],
];
```

The key (`'MyModule'`) is an arbitrary identifier. The `namespace` value must match the module's PHP namespace. Any additional keys in the array are passed as the `$config` array available inside the module as `$this->_config`.

## Module lifecycle

<Steps>
  <Step title="Constructor">
    `Module::__construct()` runs when the module is instantiated. It fires `module({ClassName}).init:before`, calls `_init()`, then fires `module({ClassName}).init:after`.
  </Step>
  <Step title="_init()">
    The module registers all its hooks, controllers, and services. This is where all module logic is wired in.
  </Step>
  <Step title="register()">
    Called earlier in the boot sequence for items that must be registered before `_init()` runs across all modules.
  </Step>
  <Step title="mapasculturais.init hook">
    Once all modules have initialized, the application fires `mapasculturais.init`. Use this hook for code that depends on all modules being ready.
  </Step>
</Steps>

## Accessing module configuration

Inside the module, configuration values are available through `$this->_config`:

```php
function _init(): void
{
    $maxItems = $this->_config['max_items'] ?? 100;
    $apiKey   = $this->_config['api_key']   ?? null;
}
```

## Real-world example: Opportunities module

The following excerpt from `src/modules/Opportunities/Module.php` shows the `_init()` pattern in practice — registering job types, hooks that run after entity operations, and hooks that intercept API parameters:

```php
function _init(): void
{
    $app = App::i();

    // Register background job types
    $app->registerJobType(new Jobs\StartEvaluationPhase(Jobs\StartEvaluationPhase::SLUG));
    $app->registerJobType(new Jobs\PublishResult(Jobs\PublishResult::SLUG));

    // Create a default registration step when a new opportunity is inserted
    $app->hook('entity(Opportunity).insert:after', function() {
        if ($this->registrationSteps && count($this->registrationSteps) > 0) {
            return;
        }
        $step = new RegistrationStep();
        $step->opportunity = $this;
        $step->save(true);
    });

    // Propagate owner changes to all subsequent phases
    $app->hook('entity(Opportunity).saveOwnerAgent', function() {
        if (!$this->isNew()) {
            foreach ($this->allPhases as $phase) {
                $phase->owner = $this->owner;
                $phase->save(true);
            }
        }
    });
}
```
