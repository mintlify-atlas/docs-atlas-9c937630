title: 'Agents API'
description: 'API endpoints for managing agents (individuals and collectives) in Mapas Culturais.'
---

An **agent** represents a cultural individual or collective registered on the platform — artists, producers, cultural managers, and similar roles. Agents are the primary ownership entity for spaces, events, projects, and opportunities.

Base path: `/api/agent/`

<Info>
  Read operations are public. Write operations (`create`, `save`, `delete`) require authentication via a JWT token. See the [Authentication guide](/auth/providers) for details.
</Info>

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/agent/find` | List or search agents |
| `GET` | `/api/agent/findOne` | Find a single agent |
| `POST` | `/agent/index` | Create a new agent |
| `PATCH` | `/agent/single/{id}` | Update an existing agent |
| `DELETE` | `/agent/single/{id}` | Delete an agent |

---

## GET /api/agent/find

Returns a paginated list of agents. Supports filtering, sorting, field selection, and full-text search.

### Query parameters

**Control parameters** (prefixed with `@`)

<ParamField query="@select" type="string">
  Comma-separated list of fields to include in the response. Use this to limit payload size. Supports dot notation for related objects (e.g., `owner.name`).

  Example: `@select=id,name,email,type`
</ParamField>

<ParamField query="@page" type="integer" default="1">
  Page number for paginated results. Starts at `1`.
</ParamField>

<ParamField query="@limit" type="integer" default="10">
  Maximum number of results per page. Use `-1` to disable pagination (use with caution on large datasets).
</ParamField>

<ParamField query="@order" type="string">
  Ordering expression. Format: `field DIR` where `DIR` is `ASC` or `DESC`.

  Example: `@order=name ASC`
</ParamField>

<ParamField query="@keyword" type="string">
  Full-text search keyword. Searches across name and description fields.
</ParamField>

<ParamField query="@or" type="integer">
  When set to `1`, combines all filter parameters with `OR` logic instead of the default `AND`.
</ParamField>

**Filter parameters**

<ParamField query="id" type="string">
  Filter by agent ID. Supports operators: `EQ(n)`, `IN(a,b,c)`, `BET(min,max)`, `!EQ(n)`.

  Example: `id=EQ(42)` or `id=BET(100,200)`
</ParamField>

<ParamField query="name" type="string">
  Filter by agent name. Supports operators: `LIKE(pattern)`, `ILIKE(pattern)` (case-insensitive).

  Example: `name=ILIKE(maria*)` matches names starting with "maria".
</ParamField>

<ParamField query="status" type="integer">
  Filter by publication status.

  | Value | Meaning |
  |-------|---------|
  | `1` | Published |
  | `0` | Draft |
  | `-2` | Archived |
  | `-9` | Disabled |
  | `-10` | Trash |

  Example: `status=EQ(1)`
</ParamField>

<ParamField query="type" type="integer">
  Filter by agent type. Common values: `1` (Individual), `2` (Collective).

  Example: `type=EQ(1)`
</ParamField>

<ParamField query="owner" type="string">
  Filter by owner agent using entity reference syntax.

  Example: `owner=EQ(@Agent:1)` returns agents owned by agent ID 1.
</ParamField>

<ParamField query="_geoLocation" type="string">
  Filter by geographic proximity. Format: `GEONEAR(longitude,latitude,radius_meters)`.

  Example: `_geoLocation=GEONEAR(-46.647,-23.541,700)`
</ParamField>

### Response

Returns a JSON array of agent objects. The fields returned depend on `@select`.

<ResponseField name="id" type="integer" required>
  Unique agent identifier.
</ResponseField>

<ResponseField name="name" type="string" required>
  Agent display name.
</ResponseField>

<ResponseField name="type" type="integer">
  Agent type. `1` = Individual, `2` = Collective.
</ResponseField>

<ResponseField name="shortDescription" type="string">
  Short description (up to 400 characters).
</ResponseField>

<ResponseField name="longDescription" type="string">
  Full long-form description.
</ResponseField>

<ResponseField name="location" type="object">
  Geographic coordinates. Only returned when `publicLocation` is true or the requester has permission to view private data.

  <Expandable title="properties">
    <ResponseField name="latitude" type="number">
      Latitude coordinate.
    </ResponseField>
    <ResponseField name="longitude" type="number">
      Longitude coordinate.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="status" type="integer">
  Publication status. See status values above.
</ResponseField>

<ResponseField name="createTimestamp" type="string">
  ISO 8601 datetime when the agent was created.
</ResponseField>

<ResponseField name="updateTimestamp" type="string">
  ISO 8601 datetime of the last update.
</ResponseField>

<ResponseField name="owner" type="object">
  The parent agent (owner). For a root agent, this is the user's profile agent.

  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      Owner agent ID.
    </ResponseField>
    <ResponseField name="name" type="string">
      Owner agent name.
    </ResponseField>
  </Expandable>
</ResponseField>

### Examples

<CodeGroup>

```bash List published agents
curl "https://instance.example.com/api/agent/find?@select=id,name,email&status=EQ(1)&@limit=20"
```

```bash Search by keyword with pagination
curl "https://instance.example.com/api/agent/find?@select=id,name,shortDescription&@keyword=teatro&@page=2&@limit=10"
```

```bash Filter by type and order by name
curl "https://instance.example.com/api/agent/find?@select=id,name,type&type=EQ(2)&@order=name%20ASC"
```

```bash Filter agents by ID range excluding one
curl "https://instance.example.com/api/agent/find?@select=id,name&id=AND(BET(100,200),!EQ(150))"
```

```bash Geo-proximity search
curl "https://instance.example.com/api/agent/find?@select=id,name,location&_geoLocation=GEONEAR(-46.6475,-23.5413,700)"
```

</CodeGroup>

## GET /api/agent/findOne

Returns a single agent. Accepts the same filter parameters as `find`. Returns the first matching result.

```bash
curl "https://instance.example.com/api/agent/findOne?@select=id,name,shortDescription&id=EQ(42)"
```

## POST /agent/index

Creates a new agent. Requires authentication.

<Warning>
  Write operations use URLs **without** the `/api/` prefix. Include a JWT token in the `authorization` header and `MapasSDK-REQUEST: true` to receive a JSON response.
</Warning>

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Agent name |
| `type` | integer | Yes | Agent type (`1` = Individual, `2` = Collective) |
| `shortDescription` | string | Yes | Short description (max 400 characters) |
| `longDescription` | string | No | Full description |
| `location` | array | No | `[longitude, latitude]` |
| `endereco` | string | No | Street address |
| `terms` | object | No | Taxonomy terms, e.g. `{"area": ["Music"]}` |

```bash
curl -X POST "https://instance.example.com/agent/index" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -d 'name=Maria da Silva&type=1&shortDescription=Atriz e produtora cultural.'
```

## PATCH /agent/single/{id}

Partially updates an existing agent. Requires authentication and ownership (or admin role).

```bash
curl -X PATCH "https://instance.example.com/agent/single/42" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true" \
  -d 'shortDescription=Updated description.'
```

---

## DELETE /agent/single/{id}

Soft-deletes an agent (moves to trash). Requires authentication and ownership (or admin role).

<Note>
  The user profile agent cannot be deleted while the user account is active.
</Note>

```bash
curl -X DELETE "https://instance.example.com/agent/single/42" \
  -H "authorization: <jwt-token>" \
  -H "MapasSDK-REQUEST: true"
```

## Status values

| Value | Constant | Meaning |
|-------|----------|---------|
| `1` | `STATUS_ENABLED` | Published and publicly visible |
| `0` | `STATUS_DRAFT` | Draft, visible only to owner |
| `-2` | `STATUS_ARCHIVED` | Archived |
| `-9` | `STATUS_DISABLED` | Disabled by admin |
| `-10` | `STATUS_TRASH` | Deleted (soft delete, in trash) |

## Tips

- Always use `@select` to specify only the fields you need. Omitting it returns the full object, which is slower.
- Use `@limit` and `@page` for paginated access. The default limit is `10`.
- Operator syntax is case-insensitive: `EQ(1)` and `eq(1)` are equivalent.
- To filter by related entity, use the `@Agent:id` syntax, e.g. `owner=EQ(@Agent:5)`.
