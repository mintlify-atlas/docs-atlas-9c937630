This guide walks you through cloning the RedeMapas repository and starting all services locally. The entire stack runs in Docker — no manual dependency installation required.

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) with Docker Compose v2 (`docker compose`, not `docker-compose`)
- [Git](https://git-scm.com/)

## Starting the local environment

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/redemapas/mapas.git
    cd mapas
    ```
  </Step>

  <Step title="Start the containers">
    ```bash
    docker compose up -d
    ```

    This starts three services: the PHP application, PostgreSQL with PostGIS, and MailPit for email capture. Database migrations run automatically on first boot via the container entrypoint.

    <Note>
      The application container waits for PostgreSQL to pass its health check before starting. On the first run, pulling images and running migrations may take a minute or two.
    </Note>
  </Step>

  <Step title="Access the application">
    Open [http://localhost:8080](http://localhost:8080) in your browser.
  </Step>

  <Step title="Check your email (optional)">
    MailPit captures all outbound email from the application. Open [http://localhost:8025](http://localhost:8025) to view captured messages — useful for testing registration flows, password resets, and notifications.
  </Step>

  <Step title="Connect to PostgreSQL (optional)">
    The database is exposed on `localhost:5432`. Connect with any PostgreSQL client using the credentials below, or use the `dev/psql.sh` wrapper script.

    | Parameter | Value |
    |---|---|
    | Host | `localhost` |
    | Port | `5432` |
    | Database | `mapas` |
    | User | `mapas` |
    | Password | `mapas-dev` |
  </Step>
</Steps>

## Services at a glance

| Service | URL | Description |
|---|---|---|
| Application | [http://localhost:8080](http://localhost:8080) | Main web interface |
| MailPit | [http://localhost:8025](http://localhost:8025) | Email capture UI |
| PostgreSQL | `localhost:5432` | Database (user: `mapas`, pass: `mapas-dev`) |

## HTTPS with Caddy (optional)

Some features — PWA installation, WebPush notifications — require HTTPS. The repository includes a `compose.override.yml` that adds a Caddy reverse proxy with a local TLS certificate.

Docker Compose loads `compose.override.yml` automatically, so no extra flags are needed:

```bash
docker compose up -d
```

<Note>
  If you previously started the stack without the override, run `docker compose down` first so Caddy's service is added cleanly.
</Note>

With the override active, two additional URLs become available:

| Service | URL |
|---|---|
| Application (HTTPS) | [https://mapas.localhost:8443](https://mapas.localhost:8443) |
| Application (HTTP redirect) | [http://mapas.localhost:8081](http://mapas.localhost:8081) |

Caddy issues a self-signed CA certificate. To make your browser trust it, run:

```bash
docker compose exec caddy caddy trust
```

<Warning>
  The `caddy trust` command modifies your operating system's trust store. On Linux you may need to follow additional steps for your specific browser — see the [Caddy documentation](https://caddyserver.com/docs/automatic-https#local-https) for details.
</Warning>

## Useful dev commands

Several wrapper scripts in the `dev/` directory execute commands inside the running containers. Run them from the repository root:

| Command | What it does |
|---|---|
| `dev/bash.sh` | Opens a shell inside the PHP application container |
| `dev/psql.sh` | Connects to PostgreSQL interactively |
| `dev/shell.sh` | Opens a PHP interactive shell (psysh) |
| `dev/pnpm.sh <cmd>` | Runs a pnpm command inside the container |

## Building frontend assets

The frontend uses pnpm workspaces. Packages live under `src/modules/`, `src/plugins/`, and `src/themes/`. Use the `dev/pnpm.sh` wrapper to run pnpm commands inside the container without needing Node.js installed locally.

```bash
# Install dependencies
dev/pnpm.sh -C src install

# Build for production
dev/pnpm.sh -C src run build

# Watch mode for development
dev/pnpm.sh -C src run watch
```

<Tip>
  Use `run watch` during active frontend development — it rebuilds assets automatically on file changes and is significantly faster than a full build.
</Tip>

## Running tests

Tests require a database connection and must run inside the application container. Each test wraps its operations in a database transaction that rolls back automatically in `tearDown`, so tests do not leave persistent state.

<CodeGroup>

```bash All tests
docker compose exec mapas vendor/bin/phpunit tests/
```

```bash Single file
docker compose exec mapas vendor/bin/phpunit tests/src/EntitiesTest.php
```

```bash Single method
docker compose exec mapas vendor/bin/phpunit tests/src/EntitiesTest.php --filter testAgentCreation
```

</CodeGroup>

<Note>
  Test helpers live in `tests/src/Abstract/TestCase.php` (base class), `tests/src/Builders/` (entity builders), and `tests/src/Directors/` (test directors). Extend `Tests\Abstract\TestCase` for new test classes.
</Note>
